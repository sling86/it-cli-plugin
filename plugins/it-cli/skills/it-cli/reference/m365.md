# Microsoft 365 Health (`m365`)

Microsoft 365 service health (Graph serviceAnnouncement) — per-service health overviews, service issues (incidents/advisories), message-centre posts. The 'is it us or Microsoft?' provider.

> Auto-generated reference. Configure: `its m365 setup`. For a command you can name, prefer live help `its m365 <resource> help` (always current) — read this file to discover what exists. [Index](./index.md)

## health

### `its m365 health overview`
Per-service health status across the tenant (Exchange, SharePoint, Teams, etc). The fast 'is it us or Microsoft?' check.
```bash
its m365 health overview
its m365 health overview --filter status=degradation
```

### `its m365 health issues`
Service incidents and advisories. Use --active to hide resolved ones, --service to scope to one service.
Flags: `--service` Filter by exact service name (e.g. Exchange Online) · `--active` Only unresolved issues
```bash
its m365 health issues
its m365 health issues --active
its m365 health issues --service "Exchange Online"
```

## messages

### `its m365 messages`
Message-centre posts — planned changes, new features, action-required items. Use --days to show only recent posts.
Flags: `--days` Only posts modified in the last N days
```bash
its m365 messages
its m365 messages --days 30
```
